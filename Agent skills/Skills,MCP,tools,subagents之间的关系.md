# MCP: "负责连接外部资源"

- 主要解决怎么拿到外部数据，外部工具，外部服务
- 可包含多个Tool
- 通常是一个接入层\标准接口层

# Tool: "负责执行具体动作"

- tool是Agent可以调用的具体能力
- 通常是一个函数，接口，API或脚本

# Subagent:"负责任务分工"

- 是一个独立工作的子代理，可以有自己的：
	- 上下文
	- 工具权限
	- 任务目标
	- 执行流程
	- 输出结果
- 解决的是复杂任务是否需要拆分给不同的角色完成

# 和Skills的关系
| 对象       | 它本身负责什么   | Skill 对它承担什么角色            |
| -------- | --------- | ------------------------- |
| MCP      | 连接外部系统和数据 | 告诉 Agent 如何理解和使用这些数据      |
| Tool     | 执行具体动作    | 告诉 Agent 什么时候用、怎么用、按什么流程用 |
| SubAgent | 分工完成任务    | 给子代理提供专业知识、行为规范和判断标准      |
